SwarWave – Privacy Policy

Effective January 2026 · Developer: Sudhir Kumar

🔒

SwarWave is privacy-first by design. We do not collect your data, we do not track you, and everything stays on your device. This page explains exactly what happens when you use the app.

1

What SwarWave Does NOT Do

  • ❌ No analytics or tracking scripts
  • ❌ No cookies set by SwarWave
  • ❌ No user accounts or registration
  • ❌ No data sent to any SwarWave server (there is none)
  • ❌ No advertising or ad networks
  • ❌ No fingerprinting
2

Data Stored Locally on Your Device

Everything is stored in your browser's localStorage or IndexedDB. If someone has physical access to your unlocked device, they may be able to view this data using browser DevTools.

Data Storage Risk
Search history / cache localStorage (v7 prefix, 24h TTL) Minimal
Favorites (videoIds) localStorage Minimal
Playlists localStorage Minimal
Recently Played localStorage Minimal
Listening Stats localStorage (plain text) Low (device only)
Offline Songs (audio blobs) IndexedDB Minimal
Theme / volume / settings localStorage None
YouTube API key (if entered) localStorage Low (your key, your device)

You can clear all local data anytime via Settings → Clear Search Cache or your browser's site data settings.

4

Security Measures

  • X-Content-Type-Options: nosniff — Prevents MIME-type sniffing attacks
  • X-Frame-Options: SAMEORIGIN — Prevents clickjacking
  • Referrer-Policy: strict-origin-when-cross-origin — Limits referrer data
  • Service Worker — Runs with limited scope, no external data exfiltration
  • No third-party scripts except YouTube IFrame API (for playback)
  • Static build — All JS/CSS inlined in a single HTML file, easy to audit
5

Android WebView (APK) Usage

  • A silent audio loop keeps the audio channel active — no data sent anywhere
  • Media Session API lets Android show lock-screen controls — handled natively by OS
  • Service Worker keeps background playback alive — local only
  • No additional tracking is added by any WebView wrapper
6

How to Clear Your Data

  1. Open the app → Settings → Clear Search Cache
  2. Open the app → Settings → Clear Offline Songs
  3. Browser: DevTools → Application → Storage → Clear Site Data
  4. Mobile: Browser Settings → Site Settings → Clear data for this site
